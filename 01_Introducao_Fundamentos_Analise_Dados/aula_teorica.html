2. Conceitos Fundamentais: Dado, Informação, Conhecimento e Tipo

Antes de qualquer ferramenta, é preciso entender a matéria-prima com a qual vamos trabalhar.

ConceitoDefiniçãoExemplo
DadoRegistro bruto, sem contexto37
TipoA natureza desse dado (número, texto, data...)37 é um número (temperatura)
InformaçãoDado organizado, com contexto"A temperatura do paciente é 37°C"
ConhecimentoInformação aplicada a uma decisão"37°C é febre? Não — está dentro do normal, não preciso agir"
💡 Pense assim: dado é a matéria-prima; informação é o produto organizado; conhecimento é a decisão que você toma com ele.

Veja isso em código — o dado vira uma variável, e o print() mostra a informação:

# DADO: um número guardado em uma variável
temperatura = 37

# INFORMAÇÃO: o dado organizado, com contexto
print(f"A temperatura do paciente é {temperatura}°C")

# CONHECIMENTO: a decisão que tomamos a partir da informação
print("37°C está dentro do normal — não preciso agir.")

3. O Ciclo de Vida da Análise de Dados

Toda análise de dados — simples ou complexa — passa por 5 etapas. Esse ciclo vai se repetir, semana após semana, em tudo que faremos no curso.

1Coleta
2Limpeza
3Análise
4Visualização
5Comunicação

5. Configurando o GitHub — Sua Pasta no Repositório da Turma

A partir de hoje, toda a turma vai trabalhar dentro de um repositório compartilhado: t5-analise-de-dados-com-python. É lá que você vai encontrar o material de cada semana e onde vai guardar (e enviar para o professor revisar) os seus próprios exercícios. Este tutorial é 100% passo a passo — siga na ordem, sem pular etapas.

📖 Antes de começar, alguns nomes que você vai ver o tempo todo:
Git
O programa que guarda o "histórico de versões" dos seus arquivos — como um Ctrl+Z gigante, que lembra de cada mudança feita, por quem e quando.
GitHub
O site que guarda esse histórico na internet (na nuvem) e permite que várias pessoas trabalhem no mesmo projeto, cada uma na sua parte.
Repositório ("repo")
A "pasta do projeto" versionada — no nosso caso, o repositório da Turma T5.
Branch
Uma "cópia paralela" do repositório, só sua, onde você trabalha sem risco de estragar o trabalho dos colegas.
Commit
Um "ponto de salvamento" das suas mudanças, com uma mensagem curta explicando o que mudou.
Push
Enviar os seus commits do seu computador para o GitHub (a nuvem) — antes do push, suas mudanças existem só na sua máquina.
Pull Request (PR)
Um pedido formal: "professor, pode revisar e aceitar essa mudança que eu fiz?" — nada entra na pasta oficial da turma sem passar por um PR aprovado.

  • Ter enviado seu nome de usuário do GitHub para o professor no Discord, para ser adicionado como colaborador do repositório (sem isso, nenhum dos dois caminhos abaixo funciona)
  • Escolher um caminho: Caminho A — GitHub Desktop (programa com botões, recomendado para quem nunca usou Git) ou Caminho B — Terminal/Git Bash (digitando comandos)

Caminho A — GitHub Desktop (recomendado para iniciantes)

  1. Criar sua conta no GitHub

    O que é: o GitHub funciona como uma "rede social" para projetos de código — você precisa de uma conta lá, assim como tem uma conta de e-mail.

    Como fazer: acesse github.com/join, informe e-mail, senha e escolha um nome de usuário (ex.: joao-silva). Esse nome de usuário é o que você vai mandar para o professor.

  2. Avisar o professor o seu usuário do GitHub e aceitar o convite

    Por quê: o professor precisa te adicionar como "colaborador" do repositório da turma. Sem isso, mesmo fazendo tudo certo nos próximos passos, você recebe erro de permissão.

    Como fazer: mande seu usuário (@seu-usuario) no Discord da turma. Você vai receber um e-mail/notificação do GitHub com um convite — abra e clique em "Accept invitation".

  3. Instalar o GitHub Desktop

    O que é: um programa com botões e telas (sem precisar digitar comandos) que conversa com o Git e o GitHub por trás dos panos.

    Como fazer: acesse desktop.github.com, baixe a versão para o seu sistema (Windows/Mac), abra o instalador e siga as opções padrão.

  4. Fazer login no GitHub Desktop

    Por quê: o programa precisa saber que é você, para associar tudo que você fizer à sua conta do GitHub.

    Como fazer: abra o GitHub Desktop → na tela inicial, clique em "Sign in to GitHub.com" → faça login com a conta criada no Passo 1.

  5. Clonar (baixar) o repositório da turma

    O que é "clonar": baixar uma cópia completa do repositório da turma para o seu computador, já conectada ao GitHub — é a partir dessa cópia que você vai trabalhar.

    Como fazer: File → Clone Repository → aba "GitHub.com" → procure t5-analise-de-dados-com-python → escolha uma pasta no seu PC → Clone.

  6. Criar a sua branch

    Por quê: trabalhar direto na main (a pasta "oficial") é bloqueado de propósito — toda mudança precisa passar por revisão. Criando sua própria branch, você tem liberdade total para trabalhar sem afetar ninguém.

    Como fazer: clique em "Current Branch" (topo) → "New Branch" → nome: alunos/seu-nome (ex.: alunos/joao-silva, sem espaços ou acentos) → Create Branch.

  7. Criar sua pasta e colocar seus arquivos

    Por quê: cada aluno trabalha isolado dentro da própria pasta em alunos/ — isso evita que um aluno sobrescreva o material de outro.

    Como fazer: no GitHub Desktop, Repository → Show in Explorer (abre a pasta clonada). Dentro de alunos/, crie uma pasta com o seu nome (igual ao usado na branch). Copie para lá os arquivos que quiser guardar.

  8. Commit — salvar uma versão com explicação

    O que é um commit: um "ponto de salvamento" das suas mudanças, junto com uma mensagem curta dizendo o que foi feito — assim, qualquer pessoa (inclusive você, no futuro) entende o histórico.

    Como fazer: volte ao GitHub Desktop — a aba "Changes" já mostra os arquivos novos. Escreva uma frase no campo "Summary" (ex.: "Adiciona meus primeiros materiais") e clique em "Commit to alunos/seu-nome".

  9. Push — enviar para o GitHub

    Por quê: até aqui, tudo ficou só no seu computador. O push é o que efetivamente manda suas mudanças para a nuvem, onde o professor e a turma conseguem ver.

    Como fazer: clique no botão "Push origin" no topo do GitHub Desktop.

  10. Abrir o Pull Request

    O que é: o pedido formal para o professor revisar e aprovar sua mudança antes dela entrar de fato na pasta oficial (main) da turma.

    Como fazer: depois do push, aparece um botão "Create Pull Request" — clique (abre o navegador), escreva um título/descrição curtos e clique em "Create pull request".

  11. Aguardar a aprovação

    Isso é normal: sua PR vai aparecer marcada como "Review required" — não é erro, é a proteção que existe para todo o material da turma. O professor revisa, e só depois disso seu conteúdo entra na pasta oficial.

Caminho B — Terminal / Git Bash

  1. Instalar o Git

    O que é: o "motor" por trás de tudo isso — o programa que de fato versiona os arquivos (o GitHub Desktop é só uma interface bonita por cima dele).

    Como fazer: acesse git-scm.com/downloads, baixe para o seu sistema, abra o instalador e siga com as opções padrão (Next, Next... Install).

  2. Abrir o Git Bash

    Como fazer: depois de instalar, procure "Git Bash" no menu Iniciar do Windows e abra — é nele que você vai digitar os comandos dos próximos passos.

  3. Configurar seu nome e e-mail (uma única vez)

    Por quê: toda vez que você salva uma versão (commit), o Git anota quem fez aquela mudança. Esse passo configura essa "assinatura" — só precisa fazer uma vez por computador.

    git config --global user.name "Seu Nome"
git config --global user.email "seu-email@exemplo.com"

  4. Clonar o repositório

    O que é "clonar": baixar uma cópia completa do repositório da turma para o seu computador.

    git clone https://github.com/cfneves/t5-analise-de-dados-com-python.git
cd t5-analise-de-dados-com-python

  5. Criar sua branch

    Por quê: a main é protegida de propósito — criar sua própria branch te dá um espaço livre para trabalhar sem afetar ninguém.

    git checkout -b alunos/seu-nome

    (troque seu-nome pelo seu nome, sem espaços/acentos — ex.: alunos/joao-silva)

  6. Criar sua pasta e adicionar arquivos

    Por quê: cada aluno trabalha isolado na própria pasta dentro de alunos/.

    mkdir alunos/seu-nome

    Copie seus arquivos para dentro dessa pasta (pelo Explorador de Arquivos mesmo, ou pelo terminal).

  7. Commit

    O que faz: add seleciona o que vai entrar no "ponto de salvamento"; commit de fato salva essa versão, com uma mensagem explicando o que foi feito.

    git add alunos/seu-nome
git commit -m "Adiciona meus primeiros materiais"

  8. Push

    O que faz: envia sua branch (com os commits) do seu computador para o GitHub — antes disso, tudo existia só na sua máquina.

    git push -u origin alunos/seu-nome

  9. Abrir o Pull Request

    O que é: o pedido formal para o professor revisar e aprovar sua mudança antes dela entrar na main oficial.

    Como fazer: o terminal mostra um link parecido com https://github.com/.../pull/new/alunos/seu-nome — copie e cole no navegador. Ou acesse o repositório no GitHub: vai aparecer um aviso amarelo "Compare & pull request" — clique nele. Escreva um título/descrição curtos e clique em "Create pull request".

  10. Aguardar a aprovação

    Isso é normal: sua PR aparece marcada como "Review required" — é a proteção que existe para o material de toda a turma. Aguarde a revisão do professor.
